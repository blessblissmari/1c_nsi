1. Цели, задачи и назначение
Создание единого функционального инструмента для работы с НСИ в области ТОиР.
Цели: 
1. Ускорить обработку и создание НСИ.
2. Снизить затраты и численность персонала для создания НСИ
3. Сформировать единую методику формирования НСИ ТОиР для разных специалистов и направлений деятельности. 
4. Создать базу для хранения и обмена данными об оборудовании, его характеристиками, спецификациями, планами ППР, методами и способами их обслуживания, ремонта и поддержания надёжности внутри компании.
5. Создать функционал по подбору аналогов для моделей и запасных частей.
6. Создать функционал по расчету параметров надежности для моделей.

Задачи и назначение:
1. Создать единую методологию формирования НСИ, классификации оборудования, компонентов, их нормализации и формирования ТК и спецификаций для них.
2. Создать единую программную среду для доступа и обмена информацией.
3. Создать и вести базу данных оборудования, классифицированную согласно единой методологии, справочники моделей и характеристик;
4. Создать и вести справочник моделей оборудования;
5. Создать и вести справочник видов воздействия и периодичностей;
6. Создать и вести справочники спецификаций на оборудование.
7. Создать и вести реестры оригиналов и аналогов моделей и запасных частей.
8. Анализировать параметры надежности модели и по компонентам (узлам и деталям).

2. Термины и определения
Иерархия - расположение разных элементов одной системы в порядке определенной логической связи
Класс - признак объекта, указывающий на его функциональные особенности. Данный справочник является основным для унификации и определения различных наборов данных, присущих определенному типу объектов.
Подкласс - признак объекта, указывающий на его конструктивные особенности.
Компонент - это состав частей объекта ремонта, структура из элементов и подэлементов
Элемент - это крупная составная часть объекта ремонта
Подэлемент - это составная часть элемента объекта ремонта
Операция ТОиР - описание одной из последовательно выполняемых задач ТОиР. ТОиР состоят из перечня последовательных операций. Основанием для выделения операции ТОиР является использование ее для целей планирования (т. е. операция ремонта может выполняться или не выполняться)
Код модели - это буквенно-цифровое обозначение объекта, который несет в себе емкую информацию о конструктивных и технических характеристиках.
3. Сокращения
НСИ - нормативная справочная информация
ТОиР - техническое обслуживание и ремонт оборудования
ТОР - типовой объект ремонта
ТМЦ - товарно-материальные ценности
ТК - технологическая карта на ТОиР
ВВ - вид воздействия
ППР - планово-предупредительный ремонт
BOM - (Bill of Materials) спецификация материалов ТМЦ
APL - (Application parts list) список компонентов для ВВ ТОиР
AOPL - (Application operation parts list) список запасных частей для операции ТОиР

6.1 Окно работы с иерархией и моделями
Окно предназначено для загрузки правил нормализации, классификации и обработке моделей оборудования. В данном окне реализована загрузка документации на типовой объект ремонта (ТОР)
Типовой объект ремонта это модель с предопределенным классом/подклассом для нее, в формате сцепки:



Входная информация:
1. Иерархия оборудования с моделями;
2. Правила нормализации НСИ;
3. Классификатор оборудования "Классы/Подклассы";

Функции:
1. Загрузка правил нормализации НСИ
2. Отображение иерархии оборудования из загруженного файла
3. Корректировка иерархии оборудования
4. Ведение справочника моделей
5. Загрузка справочника моделей
6. Ручное создание справочника моделей
7. Нормализация моделей по правилам НСИ и внесение результата в отдельное поле
8. Ведение справочника классификатора оборудования
9. Загрузка справочника классификатора оборудования
10. Классификация моделей на основании классификатора
11. Классификация неопознанных моделей оборудования (на пустые позиции из п.10)
12. Ручная корректировка правил нормализации в системе.

Функционал:
1. Кнопка "Загрузить иерархию и модели" - позволяет загрузить файл с иерархией и моделями оборудования;
2. Кнопка "Редактирование иерархии" - позволяет редактировать иерархию;
3. Кнопка "Загрузка справочника моделей" - позволяет загрузить справочник моделей;
4. Кнопка "Ручное создание справочника моделей" - позволяет создавать/заносить данные по новым моделям в справочник моделей;
5. Кнопка "Загрузить правила нормализации" - позволяет загрузить правила нормализации НСИ;
6. Кнопка "Редактирование правил нормализации" - позволяет загрузить правила нормализации НСИ;
7. Кнопка "Загрузить классификатор" - позволяет загрузить справочник Классификатор оборудования;
8. Кнопка "Нормализовать модели" - при нажатии выполняется функция нормализации всех моделей по ранее загруженным правилам, результат нормализации заноситься в поле "Код модели"
9. Кнопка "Классификация по классификатору" - при нажатии выполняется функция классификации всех моделей по ранее загруженному классификатору, заполняются поля Класс/Подкласс, непроклассифицированные модели остаются с пустыми полями класс/подкласс.
10. Кнопка "Классификация по интернет" - при нажатии выполняется функция классификации моделей и заполнения в пустые поля, если нет результата после выполнения п.9., по интернету.
11. Кнопка "Загрузить документы на ТОР" - позволяет загрузить документы на ТОР;

3. Кнопка "Загрузить справочник профессий и квалификаций" - для загрузки справочника профессий и квалификаций используемого на предприятии

5. Кнопка "Формирование справочника компонентов из источников на ТОР" - формирует для ТОР перечень его компонентов - элементы и входящие в их состав подэлементы
6. Кнопка "Обогащение справочника компонентов на ТОР из интернета" - позволяет дозаполнить разузловку и сформировать состав элементов/подэлементов используя интернет, в том случае если нет результата после выполнения п 4.
7. Кнопка "Обогащение справочника компонентов на ТОР из загруженных баз данных" - позволяет дозаполнить разузловку и сформировать состав элементов/подэлементов используя интернет, в том случае если нет результата после выполнения п 4.
8. Кнопка "Формирование перечня операций ТОиР по источнику" - выполняет поиск и привязку опериций ТОиР выполняемых и применимых к элементам/подэлементам по источнику
9. Кнопка "Обогащение операциями ТОиР по справочнику" - позволяет обогатить перечень операций ТОиР из п.6. стандартными операциями (например: очистка от грязи, протяжка резьбовых соединений и т.п.), если нет результата после выполнения п.6
10. Кнопка "Обогащение общими операциями ТОиР по интернету" - выполняется поиск и обогащение операциями ТОиР применимых к конкретному ТОР по интернету, если нет результата после выполнения п.6 и п.7
11. Кнопка "Нормализация операций" [МИС4]- выполняется общая нормализация всех внесенных операций по правилам НСИ. Действие направлено на нормализацию после п.6 и п.8. П.7 соответствует правилам НСИ изначально.
12. Кнопка "Внесение профессии/квалификации" - выполняется привязка профессии и квалификации персонала к соответствующим операциям ТОиР
13. Кнопка "Расчет трудоемкости операций на основании базы знаний" - позволяет выполнить расчет и внесение в пустые поля трудозатрат из базы знаний.
14. Кнопка "Расчет трудоемкости по интернету" - позволяет выполнить расчет и внесение в пустые поля трудозатрат по интернету, если нет результата после выполнения п.11

15. Кнопка "Формирование трудоемкости по источнику" - при совпадении операции источнику происходит внесение трудоемкости, при несовпадении/частичном совпадении (нескольких операций объединенных в одну) происходит автоматическое распределение трудозатрат с использованием ИИ
16. Кнопка "Формирование перечней ТМЦ и ед. измерения на компоненты операций ТОР из источника[МИС5]" - позволяет выполнить поиск по источнику и внести расходные ТМЦ используемые для выполнения операций ТОиР, их наименование, ед. измерения и код/номенклатуру по источнику.

8.1 Общие данные
1. Для обозначения единиц измерений должны применяться условные обозначения согласно ГОСТ 8.417 на русском языке, например: "25 м", "3.2 А", "380 В" . При переводе наименований и значений атрибутов на английский язык необходимо использовать международные обозначения единиц измерений согласно ГОСТ 8.417
2. При возникновении необходимости сокращения названий характеристик должны использоваться общеупотребимые сокращения (длина - L (или l), ширина - B (или b), высота/глубина - H (или h), толщина - s, диаметр - D (или d), радиус - R (или r), межосевое расстояние - А (или a), шаг резьбы - t, площадь - S (см. ГОСТ 2.321)
3. Диаметр внутренний обозначается малой латинской буквой "d", например: "d=5 мм". Диаметр внешний - большой латинской буквой "D", например: "D=50 мм".
4. Диаметр условного прохода и условное давление для трубопроводов и трубопроводной арматуры следует писать, как DN и PN, соответственно, например: DN32, PN16.
5. Числовые значения и их единицы измерения должны писаться через 1 пробел например: "3 м", "200 кг", "15 шт"
6. Чередующиеся размеры (длина, ширина, высота и т.п.) должны указываться через кириллический символ "дефис" без пробелов.
7. Обозначение пределов давлений указывать следующим образом: "3.2..3.5А".
8. Десятичные значения равные нулю не указываются. Вместо значения "500.000" правильно указывать "500".
9. Двойные значения указываются через слеш. Правильно указывать "220/380".

9. ЗАДАЧИ НА БУДУЩЕЕ
1. продумать внесение и закрепление отдельным этапом операций - валидные данные, например операции поверки по ГОСТу оборудования, операции и поверки утвержденные гостехнадзором (например для ГПМ, сосудов под давлением и прочее), гарантийные (например смазка по карте производителя).
2. продумать отдельным этапом дополнение информацией - справочник операций по нашим разработанным ТК, в привязке класс+подкласс+операция
3. подсвечивание операций которые обязательно должны быть в ТК (например есть операция демонтаж, но нет операции монтаж; поверка манометра; испытание оборудования на холостом ходу и прочее)
4. связь ТК с дефектами
5. вывод перечня операций, действий для устранения дефекта (автоматическое формирование ТК на устранение дефекта) и вариант ручного выбора операций для устранения дефекта (например выбор операций установкой галочек напротив нужных)
6. проработать и описать логику формирования СОП
